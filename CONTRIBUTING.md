# Contributing to Violence Detection System

Thank you for your interest in contributing to this project! We welcome contributions from the community to help improve this violence detection system.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub and include:
- A clear description of the issue.
- Steps to reproduce the bug.
- Any relevant error messages or logs.

### Suggesting Enhancements
Have an idea for a new feature or improvement? Open an issue and describe:
- The feature you'd like to see.
- Why it would be useful.
- Any implementation details you might have in mind.

### Pull Requests
1.  **Fork the repository** to your own GitHub account.
2.  **Clone the project** to your local machine.
3.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/amazing-feature
    ```
4.  **Make your changes** and commit them with clear, descriptive messages.
5.  **Push your branch** to your fork:
    ```bash
    git push origin feature/amazing-feature
    ```
6.  **Open a Pull Request** on the main repository.

## Code Style
- Please follow standard Python coding conventions (PEP 8).
- Ensure your code is well-documented with comments and docstrings.
- If you add new functionality, please try to include relevant tests (if applicable).
